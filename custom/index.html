Custom flexagonator pages

The following list describes custom pages built on top of the flexagonator toolkit aimed at different techniques and tools for working with flexagons and flexes.

  • ../index.html: General purpose UI for creating custom flexagons, applying arbitrary flex sequences, working with an interactive flexagon, creating templates, and running custom scripts.
  • atomic.html: UI for applying atomic flexes on atomic pat structure. Includes the atomic definitions of a wide variety of flexes, including morph flexes.
  • bracelet.html: UI for experimenting with running cycles of flex sequences on silver bracelets.
  • by-name.html: Categorizing and naming flexagons based on properties such as leaf shape, pat, and overall shape.
  • ennea-flip.html: UI for playing with cycles of F and F3 on an enneaflexagon (generator: (F+<<<)3).
  • explore.html: A tool for exploring all the states of a flexagon accessible from a given set of flexes. Includes options for traversing and displaying the resulting graph.
  • generator.html: Pared down interface for creating flexagons from a generating sequence and exploring the results.
  • hexa.html: Options for creating all the standard hexaflexagons with 3 to 8 faces by using generating sequences of pinch flexes. As you pinch flex around the interactive flexagon, it tracks where you are in the graph of pinch states.
  • infinity.html: Let's you easily create arbitrarily nested flexagons by providing buttons that generate the necessary pat structure for carrying out any prime flex. Interpolates face numbers and colors as it adds nested structure.
  • minimal.html: Options for seeing the minimal flexagons for some of the basic flexes.
  • morph-flexes.html: Shows how composing the morph-kite flexes are equivalent to sequences of prime flexes such as the pinch, tuck, flip, and pyramid shuffle. There are options for seeing the templates for the minimal flexagon for the morph-kite flexes.
  • multi-strip.html: A tool for splitting a template into multiple pieces, useful if it overlaps itself or if you want to break it into more manageable pieces.
  • rule110.html: Runs the 1D cellular automaton Rule 110 on a flexagon
